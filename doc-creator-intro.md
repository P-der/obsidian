# 功能简介
`doc-creator` 是一个重工业级、高度人机协同的技术文档自动化生成与排版技能（Skill）。它不只是一个“文本生成器”，而是一个“外挂架构师”。当你需要编写技术选型调研、接口说明或架构演进文档时，它会主动分析你的代码或网络资料，动态模拟相关领域的资深专家，与你进行多轮“维度确认”和“逻辑漏洞校验”，最终自动为你量身定制排版，并调用 AI 绘画能力生成专业的架构/流程配图（PNG）。

**适用场景**
- **技术调研与选型（调研类）**：例如“Vue2 升级 Vue3 与 React 18 对比”、“微服务与单体架构演进方案”。
- **项目/接口开发文档（开发类）**：基于项目现有的代码库（如 `src/api`），自动生成结构化的开发说明、README 或对外接口文档。
- **差异化受众的汇报文档**：同样的技术方案，需要分别输出给“技术总监（侧重 ROI 与宏观架构图）”或“前端开发团队（侧重时序图与代码规范）”。

**不适用场景**
- **简单的格式转换**：例如“把这段 JSON 转成 Markdown 表格”。
- **纯代码解释或问题排查**：例如“解释这段 Rust 代码是什么意思”、“帮我找找这个 Bug 的原因”。
- **与文档总结无关的纯作图**：例如“帮我画一个今年 Q3 营收数据的柱状图”。

**依赖要求**
- 本地环境中需安装/配置支持图像生成的 Skill（例如：`create-image` 或 `imagegen`，需支持 Nano Banana Pro 模型，以实现自动生成业务架构/流程图的 PNG 图片）。

# 快速开始
1. 确保 `.comate/skills/doc-creator/SKILL.md` 已在你的项目工作区或全局工作区中就绪。
2. 在 Comate 对话框中，直接抛出你的意图，即使不明确输入“文档”字眼，只要意图是总结代码或调研技术方案，即可触发。
3. 示例唤醒词：“帮我出一份关于将项目里的 Vue2 升级到 Vue3 的调研方案”、“根据 src/api 下的代码，给我写个对外的接口说明文档”。

---

# 使用方式
`doc-creator` 严格遵循“三步走”的人机协同确认工作流：
1. **阶段 1（前置思考与维度确认）**：AI 主动收集资料（网络或本地代码），思考 3-5 个审查维度，并**主动询问你是否确认这些维度**。
2. **阶段 2（深度校验与漏洞排查）**：生成初稿后，AI 会自我排查逻辑漏洞，向你**抛出校验重点和初稿摘要并请求确认**，然后进行深度调整。
3. **阶段 3（受众定制与自动配图）**：AI **询问你的目标受众（给谁看）**，随后定制排版策略，并在文档中自动嵌入生成的业务说明图（PNG）。

• **输入格式**：自然语言描述的需求或调研方向，可附带代码路径或相关参考资料。例如：“老王让我写个后端登录验证这块的接口说明文档，代码在 src/auth 目录下，你帮我总结一下写个 md。最好配个流程图。”

• **输出示例**：结构严密、分层清晰（H2/H3/列表/加粗）的 Markdown 文件，包含“整体评价”、“优化建议”、“方案对比”等专业模块，并在适当位置自动嵌入形如 `![Vue2 升级路线图](vue2_migration_diagram.png)` 的图片。

# 故障排除
- **图片未生成或显示失败**：请检查工作区中是否存在可用的 `create-image` 或 `imagegen` 技能。若因环境限制无法生成物理 PNG，AI 会自动退化为生成 Mermaid 代码块（如 `graph TD`）作为图表平替。
- **未触发技能**：请确保你的提问包含了足够的上下文（如“调研方案”、“总结这部分代码写个说明”）。如果是极其简单的提问（如“写个说明”），Comate 可能认为不需要调用重量级技能，建议增加提问细节。
- **流程中断**：由于技能包含 3 次关键的用户确认步骤，如果在提示“请问是否需要增加或调整”时你没有回复，流程将暂停。请直接回复“确认”或给出修改意见即可继续。

# 经典案例
**场景：团队技术分享，需要产出关于框架升级的调研报告**

**用户输入**：
> “我们团队打算把原先的 Vue2 升级，目前在考虑 Vue3 和 React 18。帮我出一份详细的技术调研方案。”

**执行过程（人机交互片段）**：
1. **AI (阶段 1)**：“收到。我将扮演【资深前端架构师】。针对此调研，我建议从以下 4 个维度审查：1.迁移成本、2.底层性能、3.生态兼容、4.团队 DX 体验。您看是否需要调整？”
   * **用户回复**：“确认。”
2. **AI (阶段 2)**：“初稿已出。我建议重点校验：1.是否夸大 React 学习成本；2.旧包体积风险是否被忽略。您看这部分校验重点准确吗？”
   * **用户回复**：“没问题。”
3. **AI (阶段 3)**：“请问这份文档主要给谁看？”
   * **用户回复**：“团队技术分享。”
4. **AI 最终产出**：输出名为 `vue2_migration_plan.md` 的高质量文档。文档结构以“ROI 权衡”开篇，弱化了单纯的框架优劣论；增加微前端渐进式重构方案；并自动调用作图工具，在中间插入了一张清晰的架构演进路线图（PNG）。
